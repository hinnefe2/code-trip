# Archive bot-only GitHub notification

You're being invoked on an email task that looks like a GitHub notification reporting purely automated activity. Archive it from the user's inbox.

1. **Sanity-check the body.** Confirm the body describes only bot/automation activity:
   - CI runs (GitHub Actions, CircleCI, etc.) — started / passed / failed / re-run
   - Dependabot, Renovate, or other dependency-update bots
   - Security alerts auto-generated by GitHub
   - Automated label / assignee / status changes by a GitHub App
   - Comments authored by a clearly-named bot account (suffix `-bot`, `[bot]`, or a well-known automation account)

   If you see ANY of: a human review request, a human review comment, a human approving/requesting changes, a human reply on a PR or issue thread, a human @-mention of the user — STOP and do not archive. Return: `Not archived — this involves human activity.`

2. **Archive the email.** Use `mcp__claude_ai_Gmail__unlabel_thread` with the email's `thread_id` (from the task source) and `labelIds=["INBOX"]`.

Don't ask for confirmation. When archived, return ONE sentence: `Archived GitHub bot notification: <brief subject>.`

This skill runs in two modes:
- **ACT+PTT (voice):** the user is holding the active task and explicitly asked to archive it. Their intent is the trigger.
- **Auto-handle (screener):** no user instruction; you're invoked because the screener classifier picked this skill. Apply the same body-check before archiving — when in doubt, leave it.
